A. Glossary¶

A.1. attribute¶

A.1:1 An attribute is a general, free-form metadatum that is interpreted based on its name, convention, language, and tool.

A.2. abstract data type¶

A.2:1 An abstract data type is a collection of other types.

A.3. access expression¶

A.3:1 An access expression is an expression which specifies accessing either a item within a namespace, or a field of a value.

A.4. argument operand¶

A.4:1 An argument operand is an operand which is used as an argument in a call expression

A.5. binary literal¶

A.5:1 A binary literal is a integer literal that is in base 2.

A.37:1 See BinaryLiteral.

A.6. block comment¶

A.6:1 A block comment is a comment that spans one or more lines.

A.7. bool¶

A.7:1 bool is a type whose values denote the truth values of logic and Boolean algebra.

A.8. boolean literal¶

A.8:1 A boolean literal is a literal that denotes the truth values of logic and Boolean algebra.

A.8:2 See BooleanLiteral.

A.9. built-in attribute¶

A.9:1 A built-in attribute is a language-defined attribute.

A.10. byte literal¶

A.10:1 A byte literal is a literal that denotes a fixed byte value.

A.11. call expression¶

A.11:1 A call expression is an expression that invokes a function or constructs a struct value or enum variant value.

A.12. char¶

A.12:1 char is a type whose values denote a single Unicode character.

A.13. character literal¶

A.13:1 A character literal is a literal that denotes a fixed Unicode character.

A.13:2 See CharacterContent.

A.14. comment¶

A.14:1 A comment is a lexical element that acts as an annotation to the program source, and is ignored by the compiler.

A.14:2 See Comment.

A.15. construct¶

A.15:1 A construct is a piece of program text that is an instance of a syntactic category.

A.16. decimal literal¶

A.16:1 A decimal literal is a integer literal that is in base 10.

A.17. entity¶

A.17:1 An entity is a construct that can be referred to within the program text, usually via a access expression.

A.18. escaped character¶

A.18:1 An escaped character is the textual representation for a character with special meaning. An escaped character consists of character 0x5C (\) followed by the single character encoding of the special meaning character. For example, \n is the escaped character for the newline character.

A.19. expression¶

A.19:1 An expression is a construct that produces a value, and may have side effects at run-time.

A.20. exported function¶

A.20:1 An exported function is an export of a function.

A.21. f32¶

A.21:1 f32 is a floating point type equivalent to the IEEE 754-2008 binary32 type.

A.22. f64¶

A.22:1 f64 is a floating point type equivalent to the IEEE 754-2008 binary64 type.

A.23. field¶

A.23:1 A field is an element of an abstract data type.

A.24. float literal¶

A.24:1 A float literal is a numeric literal that denotes a fractional number.

A.25. float suffix¶

A.25:1 A float suffix is a component of a float literal that specifies the explicit floating point type.

A.26. floating point type¶

A.26:1 A floating point type is a numeric type that can represent values denote fractional numbers.

A.27. function¶

A.27:1 A function is a value of a function type that models a behaviour.

A.28. function type¶

A.28:1 A function type is a unique type that identifies a function.

A.29. hexadecimal literal¶

A.29:1 A hexadecimal literal is a integer literal that is in base 16.

A.30. i8¶

A.30:1 i8 is a signed integer type whose values range from - (27) to 27 - 1, all inclusive.

A.31. i16¶

A.31:1 i16 is a signed integer type whose values range from - (215) to 215 - 1, all inclusive.

A.32. i32¶

A.32:1 i32 is a signed integer type whose values range from - (232) to 232 - 1, all inclusive.

A.33. i64¶

A.33:1 i64 is a signed integer type whose values range from - (263) to 263 - 1, all inclusive.

A.34. i128¶

A.34:1 i128 is a signed integer type whose values range from - (2127) to 2127 - 1, all inclusive.

A.35. ibig¶

A.35:1 An ibig is a signed integer type which can represent an unbounded range of whole numbers.

A.36. identifier¶

A.36:1 An identifier is a lexical element that refers to a name.

A.37. integer literal¶

A.37:1 An integer literal is a numeric literal that denotes a whole number.

A.38. inferred type¶

A.38:1 An inferred type is a type that is deduced from type inference.

A.39. integer type¶

A.39:1 A integer type is a numeric type whose values denote whole numbers.

A.40. integer suffix¶

A.40:1 An integer suffix is a component of an integer literal that specifies the explicit integer type.

A.40:2 See IntegerSuffix.

A.41. isize¶

A.41:1 isize is a signed integer type with the same number of bits as the platform’s pointer type, and is at least 16 types.

A.42. item¶

A.42:1 An item is a top level statement in a source file.

A.43. keyword¶

A.43:1 A keyword is a word in program text that has special meaning.

A.44. line¶

A.44:1 A line is a sequence of zero or more characters followed by an end of line.

A.45. line comment¶

A.45:1 A line comment is a comment that spans over one line.

A.46. literal¶

A.46:1 A literal is a fixed value in a program source.

A.47. main function¶

A.47:1 A main function is a function that acts as an entry point into a program.

A.48. module¶

A.48:1 A module is a construct that contains zero or more items.

A.49. module attribute¶

A.49:1 A module attribute is an attribute which can be specified at the module level.

A.50. name¶

A.50:1 A name is an identifier that refers to an entity.

A.51. namespace¶

A.51:1 A namespace is a logical grouping of names such that the occurrence of a name in one namespace does not conflict with an occurrence of the same name in another namespace.

A.52. numeric literal¶

A.52:1 A numeric literal is a literal that denotes a number.

A.53. numeric type¶

A.53:1 A numeric type is a type whose values denote numbers.

A.54. octal literal¶

A.54:1 An octal literal is a integer literal that is in base 8.

A.55. operand¶

A.55:1 An operand is an expression nested within an expression.

A.56. panic¶

A.56:1 A panic is an abnormal program state caused by invoking the prelude defined function.

A.57. pointer type¶

A.57:1 A pointer type is a type that represents memory locations.

A.58. prelude¶

A.58:1 The prelude is a module which is loaded into every Hash program before the main text body of a module. It defines core language items which are required for the language and accompanying tools.

A.59. punctuator¶

A.59:1 A punctuator is a character or a sequence of characters in the Punctuation category.

A.60. separator¶

A.60:1 A separator is a character or a string that separates adjacent lexical elements.

A.61. sequence type¶

A.61:1 A sequence type represents a sequence of elements.

A.62. signed integer type¶

A.62:1 A signed integer type is an integer type that can represent values denote negative whole numbers, zero, and positive whole numbers.

A.63. slice¶

A.63:1 A slice is a value of a slice type.

A.64. slice type¶

A.64:1 A slice type is a sequence type that provides a view into a sequence of elements.

A.65. source file¶

A.65:1 A source file contains the program text of module attributes, and items.

A.66. str¶

A.66:1 A str is a sequence type that represents a slice of 8-bit unsigned bytes.

A.67. string literal¶

A.67:1 A string literal is a sequence of Unicode characters enclosed in double quotes ".

A.67:2 See StringContent .

A.68. suffixed float¶

A.68:1 A suffixed float is a float literal with a float suffix.

A.69. suffixed integer¶

A.69:1 A suffixed integer is an integer literal with an integer suffix.

A.70. syntactic category¶

A.70:1 A syntactic category is a nonterminal in the Backus-Naur Form grammar definition of the Hash programming language.

A.71. type¶

A.71:1 A type defines a set of values and a set of operations that act on those values.

A.72. type inference¶

A.72:1 Type inference is the deduction of an expected type of an arbitrary value.

A.73. u8¶

A.73:1 u8 is an unsigned integer type whose values range from 0 to 28 - 1, all inclusive.

A.74. u16¶

A.74:1 u16 is an unsigned integer type whose values range from 0 to 216 - 1, all inclusive.

A.75. u32¶

A.75:1 u32 is an unsigned integer type whose values range from 0 to 232 - 1, all inclusive.

A.76. u64¶

A.76:1 u64 is an unsigned integer type whose values range from 0 to 264 - 1, all inclusive.

A.77. u128¶

A.77:1 u128 is an unsigned integer type whose values range from 0 to 2128 - 1, all inclusive.

A.78. ubig¶

A.78:1 An ubig is a unsigned integer type which can represent zero, and unbounded positive whole numbers.

A.79. unicode¶

A.79:1 Unicode is the colloquial name for the ISO/IEC 10646:2017 Universal Coded Character Set standard.

A.80. unsigned integer type¶

A.80:1 A unsigned integer type is an integer type that can represent values denote zero and positive whole numbers.

A.81. unsuffixed float¶

A.81:1 A unsuffixed float is a float literal without a float suffix.

A.82. unsuffixed integer¶

A.82:1 A unsuffixed integer is an integer literal without an integer suffix.

A.83. usize¶

A.83:1 usize is a unsigned integer type with the same number of bits as the platform’s pointer type, and is at least 16 types.

A.84. value¶

A.84:1 A value is either a literal or the result of a computation, that may be stored in a memory location, and interpreted based on a type.
